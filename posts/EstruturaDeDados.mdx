# O que são Estruturas de Dados?
Uma estrutura de dados na ciencia da computação é como se fosse uma caixa que armazena dados e tambem uma coleção de operações.

* Esses dados podem ser de qualquer tipo, sejam numeros, cadeias de caracteres ou até mesmo outras estruturas de dados. 
* Os dados podem ser armazenados de diversas formas, como por exemplo em um vetor, em uma lista, em uma pilha, em uma fila, em uma arvore e em um grafo (Não iremos entrar em detalhes sobre grafos e arvores, deixaremos pra outro momento).
* As operações que conseguimos fazer são diversas, como por exemplo, adicionar um dado, remover um dado, buscar um dado, ordenar os dados, etc. 

# Por que estudar Estruturas de Dados?
Agora por que queremos estudar Estruturas de Dados? Por que elas são tão importantes e necessarias?
primeiro de tudo elas são importantes por que nos ajudam a resolver problemas de forma mais eficiente.

Por exemplo, se temos um estrutura com 1000 elementos e queremos encontrar um elemento especifico dentro desse estrutura, podemos fazer isso de duas formas:,

1. Percorrendo o estrutura inteiro e comparando cada elemento com o elemento que queremos encontrar.
2. Ordenando o estrutura e depois usando uma busca binaria (Não abordaremos aqui) para encontrar o elemento que queremos.

A primeira forma é mais lenta e a segunda é mais rapida, por que a segunda forma é mais rapida? Por que usamos uma estrutura de dados para resolver o problema.

# Principais Tipos de Estruturas de Dados
O conceito chave aqui é saber ponteiros, se você não sabe o que são ponteiros, recomendo que você estude sobre eles antes de continuar.

## 1. Array
É a estrutura de dados mais simples e mais utilizada, é uma coleção de dados do mesmo tipo, que são armazenados em posições de memoria.
Acessamos os dados atraves de indices, que são numeros inteiros que representam a posição de um elemento dentro do array.
Uma curiosidade é que a maiora das lingaugens de programação define o indice inicial do array como 0.

### Possuimos dois tipos de arrays:

- Arrays Unidimensionais (que seguem apenas uma linha, chamados tambem de vetores)
- Arrays Multidimensionais (que seguem mais de uma linha, chamados tambem de matrizes)
<br/>

- Vantagens:
    
    - Acesso rapido aos elementos
    - Facil de implementar

- Desvantagens:
    
    - Tamanho fixo
    - O array é homogêneo, ou seja, só pode armazenar dados do mesmo tipo
    - A inserção e remoção não são fáceis arrays

# 2. Pilhas(Stacks)

A pilha é baseada no princípio LIFO (Last In First Out), ou seja, o ultimo elemento a entrar é o primeiro a sair.

Pilhas tem como fundamento duas operações basicas:

* Push: Adiciona um elemento no topo da pilha
* Pop: Remove o elemento do topo da pilha
    
Imagina-se uma pilha de pratos, onde o ultimo prato a ser colocado é o primeiro a ser retirado, por exemplo:

- Só se consegue "inserir" (Que no caso seria empilhar) novos pratos no topo da pilha.

- Só se consegue "remover" (Que no caso seria desempilhar) pratos do topo da pilha.

- Vantagens:

    - Acesso rapido aos elementos
    - Facil de implementar

- Desvantagens:

    - A pilha é homogênea, ou seja, só pode armazenar dados do mesmo tipo
    - A pilha tambem tem acesso restrito, ou seja, só podemos acessar o elemento do topo da pilha
    - A pilha não é dinamica, ou seja, não pode crescer ou diminuir

<br/>
#### Codigo de uma pilha que insere e remove dados;
<br/>

```c
#include <stdio.h>
#include <stdlib.h>
#define MAX 10
typedef struct{
    int topo;
    int vet[MAX];
}pilha;
void inicializa(pilha *p){
    p->topo = -1;
}
void push(pilha *p, int x){
    if(cheia(p)){
        printf("Pilha cheia\n");
        return;
    }
    p->topo++;
    p->vet[p->topo] = x;
}
int pop(pilha *p){
    if(vazia(p)){
        printf("Pilha vazia\n");
        return -1;
    }
    int aux = p->vet[p->topo];
    p->topo--;
    return aux;
}
```

# 3. Filas(Queues)

São similares ás pilhas, as filas são baseadas no princípio FIFO (First In First Out), ou seja, o primeiro elemento a entrar é o primeiro a sair.

Filas tem como fundamento duas operações basicas:

* Enqueue: Adiciona um elemento no final da fila
* Dequeue: Remove o elemento do inicio da fila

Imagina-se uma fila de pessoas, onde a primeira pessoa a entrar é a primeira a sair.

- Só se consegue "inserir" (Que no caso seria enfileirar) novas pessoas no final da fila.
- Só se consegue "remover" (Que no caso seria desenfileirar) pessoas do inicio da fila.

- Vantagens:
    - Acesso rapido aos elementos
    - Facil de implementar

- Desvantagens:
    - A fila é homogênea, ou seja, só pode armazenar dados do mesmo tipo
    - A fila tambem tem acesso restrito, ou seja, só podemos acessar o elemento do inicio da fila
    - A fila não é dinamica, ou seja, não pode crescer ou diminuir


<br/>
#### Codigo de uma fila que insere e remove dados;
<br/>

```c
#include <stdio.h>
#include <stdlib.h>
#define MAX 10
typedef struct{
    int inicio;
    int fim;
    int vet[MAX];
}fila;
void inicializa(fila *f){
    f->inicio = 0;
    f->fim = 0;
}
void enqueue(fila *f, int x){
    if(cheia(f)){
        printf("Fila cheia\n");
        return;
    }
    f->vet[f->fim] = x;
    f->fim++;
}
int dequeue(fila *f){
    if(vazia(f)){
        printf("Fila vazia\n");
        return -1;
    }
    int aux = f->vet[f->inicio];
    f->inicio++;
    return aux;
}
```

# 4. Listas Ligadas(Linked Lists)

Já as listas são estruturas de dados dinamicas, ou seja, elas podem crescer ou diminuir de tamanho
mas a sua maior diferença é que os elementos não precisam estar em posições de memoria contiguas, ou seja, os elementos podem estar em qualquer lugar da memoria.

Como funcionam as listas ligadas em si? Bom, cada elemento da lista é chamado de nó, e cada nó tem dois campos, um campo que armazena o dado e um campo que armazena o endereço do proximo nó.

Pra que elas são utilzadas? Bom, elas são utilizadas quando não sabemos o tamanho da lista, ou seja, quando não sabemos quantos elementos a lista vai ter, ou quando não sabemos o tamanho maximo que a lista pode ter.


Listas tem como fundamento duas operações basicas:

* Insert: Adiciona um elemento em uma posição especifica da lista
* Remove: Remove um elemento de uma posição especifica da lista

- Vantagens:
    - Acesso rapido aos elementos
    - Facil de implementar
    - A lista é dinamica, ou seja, pode crescer ou diminuir

- Desvantagens:
    - A lista ligada é homogênea, ou seja, só pode armazenar dados do mesmo tipo
    

<br/>
#### Codigo de uma lista ligada que insere e remove dados;
<br/>

```c
#include <stdio.h>
#include <stdlib.h>
typedef struct no{
    int dado;
    struct no *prox;
}no;
typedef struct{
    no *inicio;
}lista;
void inicializa(lista *l){
    l->inicio = NULL;
}
void insere(lista *l, int x){
    no *novo = (no*)malloc(sizeof(no));
    novo->dado = x;
    novo->prox = l->inicio;
    l->inicio = novo;
}
void remove(lista *l, int x){
    no *aux = l->inicio;
    no *ant = NULL;
    while(aux != NULL && aux->dado != x){
        ant = aux;
        aux = aux->prox;
    }
    if(aux == NULL){
        printf("Elemento nao encontrado\n");
        return;
    }
    if(ant == NULL){
        l->inicio = aux->prox;
    }else{
        ant->prox = aux->prox;
    }
    free(aux);
}

```
